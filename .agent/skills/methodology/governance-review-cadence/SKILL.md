# Governance Review Cadence

## Description

Standardizing periodic governance, risk, and compliance review cycles for technical and agent systems.

## Implementation Instructions

1. Schedule review intervals based on ecosystem risk (e.g., Weekly for Trading, Monthly for Creative).
2. Define the "Governance Pack": Policy violations, Risk scores, Security audits.
3. Conduct 30m review sessions with relevant agent/human stakeholders.
4. Log all decisions and policy updates into the Shared Brain.
5. Track remediation tasks from the review to completion.

## Constraints

- **DO NOT** skip governance reviews because "nothing changed."
- **ALWAYS** maintain a non-repudiable log of review attendees and decisions.
- **DO NOT** allow p0 findings to persist beyond the next review cycle.
