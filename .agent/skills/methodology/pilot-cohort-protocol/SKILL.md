# Pilot Cohort Protocol

## Description

Pilot cohort design and iteration — recruiting, running, and extracting curriculum improvement data from a beta student group.

## Implementation Instructions

1. Define pilot cohort size: 10-20 students for meaningful feedback without noise.
2. Set pilot goals: completion rate target, NPS benchmark, and top-3 issues to identify.
3. Recruit pilot students via wait-list, community, or direct invitation.
4. Run a live cohort with weekly check-ins — collect mid-course NPS and structured feedback.
5. Analyse completion rates and dropout points: modules with over 30% drop-off need redesign.
6. Synthesise findings into a Curriculum Iteration Report before full launch.

## Constraints

- DO NOT launch publicly without at least one pilot cohort completed.
- ALWAYS compensate or discount pilot participants — they are paying with their time.
- DO NOT ship a module with over 30% dropout rate without redesigning it first.
