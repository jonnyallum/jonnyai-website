# Weak Signal Tracking

## Description

Early-detection system for tracking fringe trends, emerging technologies, or subtle market shifts before they enter the mainstream.

## Implementation Instructions

1. Define "radar" sources (fringe blogs, tech forums, deep-web research).
2. Set alerts for low-volume but high-acceleration keywords.
3. Correlate weak signals with historical trend birth-patterns.
4. Classify signals into "Watch," "Incubate," or "Discard" lists.
5. Report emerging opportunities to product and strategy teams.

## Constraints

- **DO NOT** wait for mainstream validation before reporting a signal.
- **ALWAYS** separate the "signal" (event) from the "noise" (hype).
- **DO NOT** discard signals simply because volume is low.
