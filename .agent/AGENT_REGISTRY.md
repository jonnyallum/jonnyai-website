## 📏 The Build Standard

Every completed agent MUST have ALL of the following. No exceptions.

### Required Structure (29 elements)

1. YAML frontmatter (name, description, tier, allowed_tools)
2. H1: `# [Human Name] - Agent Profile`
3. Philosophy Quote: `> _"..."_`
4. `---` separator
5. `## The Creed` (identical, unmodified)
6. `---` separator
7. `## Identity` table (7 rows: Handle, Human Name, Nickname, Role, Authority Level, Accent Color, Signs Off On)
8. `---` separator
9. `## Personality` (4 subsections: Vibe, Communication Style, Working Style, Quirks)
10. `---` separator
11. `## Capabilities`
12. `### Can Do ✅` (5+ items, each with bold name + specific deliverable)
13. `### Cannot Do ❌` (3+ items, each naming the agent who handles it instead)
14. `### Specializations 🎯` (table with Domain, Expertise Level, Notes)
15. `---` separator
16. `## Standard Operating Procedures`
17. Minimum 3 SOPs, ideally 4 (SOP-001 through SOP-004)
18. Each SOP: H3 title, **Trigger:** line, numbered steps
19. `---` separator
20. `## Collaboration`
21. `### Inner Circle` table (3-5 agents, with Handoff Pattern direction + content)
22. `### Reports To` (always @marcus)
23. `### Quality Gates` table
24. `---` separator
25. `## Feedback Loop` (Before 3 steps / After 3 steps)
26. `---` separator
27. `## Performance Metrics` table (Task completion + 3 domain-specific + Shared Brain sync)
28. `---` separator
29. `## Restrictions` (Do NOT ❌ 5+ / ALWAYS ✅ 5+)
30. `---` separator
31. `## Tools & Resources` (Primary Tools + MCP Servers)
32. `---` separator
33. `## Learning Log` (empty table)
34. `---` separator
35. `## 📜 Governing Directives` (identical, unmodified)
36. `---` separator
37. Footer: `_Jai.OS 5.0 | The Antigravity Orchestra | Last Updated: [date]_`

## ✅ Dos — Building a Great Agent

- **DO** run the soul test on every Philosophy Quote and Vibe
- **DO** write SOPs with precise Triggers that name who fires them and under what condition
- **DO** make every SOP step start with a verb that implies a specific action
- **DO** name the receiving agent in every final SOP step
- **DO** make every Handoff Pattern show direction (→) and specific content
- **DO** include what frustrates AND energises the agent in the Vibe
- **DO** give Quirks that are specific behaviours, not adjectives
- **DO** use domain-specific performance metrics with real targets (numbers, not "high")
- **DO** run Ralph Loop validation before declaring any build complete
- **DO** update AGENT_REGISTRY.md after every completed build
- **DO** read `perfect_agent_template.md` before every build — never rely on memory

## ❌ Don'ts — What Ruins an Agent

- **DON'T** copy-paste SOPs from one agent to another — every SOP must reference domain-specific data
- **DON'T** use these phrases anywhere: "passionate about", "strives to deliver", "works collaboratively", "high-quality outputs", "dedicated professional"
- **DON'T** write a Cannot Do without naming who handles it instead
- **DON'T** write Quirks that are just adjectives ("meticulous", "detail-oriented")
- **DON'T** leave the Philosophy Quote as something that could describe any other agent
- **DON'T** use generic performance metrics ("Output quality: High")
- **DON'T** ship without running `validate_agents.py` with zero errors
- **DON'T** leave Legacy AgOS 2.0 sections (🎭 Persona Overview, Knowledge Base / Context, Personal Development Plan) in any file
- **DON'T** write Handoff Patterns as "works with @[agent]" — always show direction and content
- **DON'T** invent domain knowledge — build the structure, populate from the brief or Shared Brain

## 🔍 External AI Audit Checklist

When another AI (Gemini, GPT, Claude instance) submits a SKILL.md for review:

1. Run `validate_agents.py` — capture all errors and warnings
2. Run Perfect Agent Quality Checklist (all 37 items from `perfect_agent_template.md`)
3. Grade each failure: CRITICAL / HIGH / MEDIUM / LOW
4. Check specifically for:
   - Philosophy Quote that passes the soul test (could it describe any other agent?)
   - SOPs with vague steps ("analyse the data") vs. executable steps
   - Handoff Patterns with direction and content
   - Performance metrics with real numbers
   - Quirks that are behaviours, not adjectives
   - Cannot Do items that name the routing agent
5. Produce audit report: Grade (Pass/Conditional Pass/Fail), failure list, remediation instructions
6. Post audit result to chatroom as DSP
