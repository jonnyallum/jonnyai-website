## Detailed Training Day Skill Breakdown

### Conductor – Orchestrator

| Skill                             | Description                                                                                                |
| :-------------------------------- | :--------------------------------------------------------------------------------------------------------- |
| **Skills matrix + gap detection** | Identifies which agents need upskilling based on recent failures and overlaps.                             |
| **Continual-learning schedule**   | Plans periodic "retraining" sessions for agents based on new data, docs, and incidents.                    |
| **Cross-agent drills**            | Runs fire-drill scenarios (outage, feature launch, big spike) to test collaboration and improve playbooks. |

### Forge – Idea → Money

| Skill                      | Description                                                                                                 |
| :------------------------- | :---------------------------------------------------------------------------------------------------------- |
| **Offer iteration engine** | Auto-generates new offers (bonuses, guarantees, bundles) when conversion lags, then passes tests to Metric. |
| **Portfolio thinking**     | Designs multiple monetisation tracks (short-term cash vs long-term equity/value) for the same idea.         |
| **Segment pivoting**       | Proposes adjacent segments and quick tests when a primary niche doesn't pay.                                |

### Helpline – Support & Success

| Skill                              | Description                                                                                        |
| :--------------------------------- | :------------------------------------------------------------------------------------------------- |
| **Playbook learning from tickets** | Promotes recurring "workarounds" into documented plays and asks Archivist/Echo to formalise them.  |
| **Churn pattern spotting**         | Works with Metric to spot early warning patterns and auto-launch save plays.                       |
| **Agent coaching**                 | Creates "what to say" patterns from the best human replies and shares them with Echo/other agents. |

### Jonny AI – Development

| Skill                            | Description                                                                                                 |
| :------------------------------- | :---------------------------------------------------------------------------------------------------------- |
| **Production-aware coding**      | Adapts patterns based on Sentinel's bug reports and Vaultguard's findings (lessons baked into future code). |
| **Progressive refactor planner** | Plans refactors in small safe steps tied to tests and metrics, not big-bang rewrites.                       |
| **Cross-stack templates**        | Reusable blueprints for "CRUD + auth + analytics + billing" that respect Metric/Forge/Datastore needs.      |

### Sentinel – QA / Testing

| Skill                       | Description                                                                                                    |
| :-------------------------- | :------------------------------------------------------------------------------------------------------------- |
| **Flaky test hunter**       | Identifies tests that frequently fail without code changes, quarantines, and suggests refactors.               |
| **Risk-based testing**      | Weights tests against monetisation and user impact (Forge/Metric signals) to prioritise what must never break. |
| **Learning from incidents** | Auto-creates regression tests from real production incidents.                                                  |

### Vaultguard – Security

| Skill                             | Description                                                                                           |
| :-------------------------------- | :---------------------------------------------------------------------------------------------------- |
| **Threat memory**                 | Maintains a memory of past security incidents and near-misses to update checklists and code patterns. |
| **Business-aligned risk scoring** | Ties risk levels to revenue and legal impact (signals from Forge/Metric).                             |
| **Security play dry-runs**        | Runs breach tabletop exercises with Conductor/Sentinel/DevOps.                                        |

### DevOps – Infra / CI/CD

| Skill                           | Description                                                                                 |
| :------------------------------ | :------------------------------------------------------------------------------------------ |
| **Self-tuning CI**              | Adjusts parallelism, caching and test selection based on historical runs and fail patterns. |
| **Cost-performance dashboards** | Auto-summaries tying infra cost to revenue and usage (with Metric).                         |
| **Rollback drills**             | Scheduled practice rollbacks with Sentinel and Vaultguard.                                  |

### Datastore – DB / Supabase

| Skill                                  | Description                                                                      |
| :------------------------------------- | :------------------------------------------------------------------------------- |
| **Workload-aware indexing**            | Periodically proposes/drops indexes based on real query stats and Realtime load. |
| **Tenant-level performance analytics** | Identifies which customer segments hammer the DB and informs Forge/Metric.       |
| **Schema evolution advisor**           | Suggests safer patterns when repeated migration pain appears.                    |

### Metric – Analytics & Insights

| Skill                      | Description                                                                                    |
| :------------------------- | :--------------------------------------------------------------------------------------------- |
| **Auto-KPI suggestions**   | Proposes new metrics when it sees consistent patterns (e.g. upgrade path, feature stickiness). |
| **Anomaly → play linkage** | When spotting anomalies, suggests specific actions for Forge, Helpline, Autoflow.              |
| **Analytics debt tracker** | Surfaces untracked flows or dirty models that block good decisions.                            |

### Autoflow – Automation Engineer

| Skill                         | Description                                                                  |
| :---------------------------- | :--------------------------------------------------------------------------- |
| **Self-audit of automations** | Flags brittle zaps/flows (high failure rate, manual overrides) for redesign. |
| **Learning from manual work** | Spots repeated manual actions in logs and proposes automations.              |
| **Safety-aware automation**   | Tags flows by risk level and requires human approvals on high-risk paths.    |

### Goldie – SEO / Growth

| Skill                       | Description                                                                |
| :-------------------------- | :------------------------------------------------------------------------- |
| **Backlink-quality memory** | Learns which link sources and content types actually move ranking/traffic. |
| **Commercial-intent radar** | Prioritises keywords that tie closest to paid actions in Metric.           |
| **SERP experiment engine**  | Systematically tests titles/meta/angles and records winners as templates.  |

### Clippers – Video / Viral

| Skill                      | Description                                                                         |
| :------------------------- | :---------------------------------------------------------------------------------- |
| **Hook library evolution** | Keeps a live library of top-performing hooks/format patterns by niche and platform. |
| **Watch-through learning** | Adapts pacing and structure based on watch time/retention curves from Metric.       |
| **Repurposing brain**      | Auto-maps each winning video into shorts, carousels, email angles for Echo.         |

### Pixel – UI/UX

| Skill                        | Description                                                                          |
| :--------------------------- | :----------------------------------------------------------------------------------- |
| **UX pattern memory**        | Catalogues which patterns reduce friction based on Helpline tickets and Metric data. |
| **Design experiments queue** | Constantly lines up small UX A/B tests for key flows (checkout, onboarding).         |
| **Accessibility coach**      | Nudges Jonny/Echo with patterns learned from Sentinel's a11y checks.                 |

### Echo – Content & UX Copy

| Skill                     | Description                                                                     |
| :------------------------ | :------------------------------------------------------------------------------ |
| **Conversion phrasebook** | Systematically logs top-performing headlines/CTAs/error messages by context.    |
| **Tone-shift training**   | Better modulation by situation (escalations, incidents, legal-sensitive comms). |
| **Support-aware copy**    | Rewrites in-product text that keeps generating Helpline tickets.                |

### Archivist – Docs / Knowledge

| Skill                            | Description                                                                                            |
| :------------------------------- | :----------------------------------------------------------------------------------------------------- |
| **Skills matrix + curriculum**   | Keeps a skills matrix for each agent and proposes upskilling "courses" (new prompts, tools, datasets). |
| **Incident-to-doc pipeline**     | Every major issue automatically becomes or updates a runbook.                                          |
| **Redundancy & overlap mapping** | Identifies conflicting knowledge across agents and resolves it.                                        |

### Manus – MCP / Augmentation

| Skill                          | Description                                                              |
| :----------------------------- | :----------------------------------------------------------------------- |
| **Discovers new MCP tools**    | Actively monitors the MCP registry for new and trending servers.         |
| **Tests integration patterns** | Prototypes new server integrations before committing to the main config. |
| **Extends agency reach**       | Connects the agency to revenue-generating APIs and external systems.     |
| **Audits community servers**   | Works with Sentinel to verify security before onboarding a new server.   |

### Nathan – Automation & Outreach

| Skill                          | Description                                                                                        |
| :----------------------------- | :------------------------------------------------------------------------------------------------- |

| **Email Pipeline Diagnostics** | Uses Resend API for inbound/outbound health checks and automated warmup.                           |
| **Milestone Logic**            | Auto-detects project milestones from the Shared Brain to trigger social broadcasts.                |

| Skill                          | Description                                                                               |
| :----------------------------- | :---------------------------------------------------------------------------------------- |

| Skill                    | Description                                                                      |
| :----------------------- | :------------------------------------------------------------------------------- |

| Skill                        | Description                                                                                |

### Delegator – Meta-Orchestrator (@delegator)

| Skill                     | Description                                                                            |
| :------------------------ | :------------------------------------------------------------------------------------- |

| **Bottleneck Prediction** | Predicts which agent dependency chains will stall and proactively re-routes tasks.     |
| **State Packet Auditing** | Verifies the quality of cross-agent handoffs via Deterministic State Packets (DSPs).   |
