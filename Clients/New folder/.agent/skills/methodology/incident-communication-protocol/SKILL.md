# Incident Communication Protocol

## Description

Structured methodology for stakeholder communication, status updates, and reporting during and after technical incidents.

## Implementation Instructions

1. Define communication roles (Commander, Liaison, Scribe).
2. Set up a central incident channel for internal updates.
3. Establish regular update intervals (e.g., every 30m) for stakeholders.
4. Use standardized templates for "In Progress," "Stable," and "Resolved" statuses.
5. Publish a professional Postmortem within 24 hours of resolution.

## Constraints

- **DO NOT** leave stakeholders in silence for longer than 60m during an outage.
- **ALWAYS** prioritize accuracy over speed in public-facing updates.
- **DO NOT** speculatively assign blame in public status reports.
